# Student Course Allocation Automation System

Automates university elective allocation based on student CGPA priority, course preferences, seat availability, prerequisites, and section balancing.

---

## Project Structure

```
course_allocation/
├── main.py                  ← entry point (CLI)
├── web_app.py               ← entry point (Web UI)
├── input_processor.py       ← load, clean, validate student data
├── section_manager.py       ← seat tracking & section balancing
├── course_allocator.py      ← G1/G2 allocation engine + section swap
├── report_generator.py      ← Excel report builder
├── create_sample_data.py    ← generate config.xlsx & responses.xlsx for testing
├── requirements.txt
├── config.xlsx              ← course configuration (generated by sample script)
├── responses.xlsx           ← Google Form export (your upload or sample)
├── clean_students.json      ← intermediate cleaned data (auto-generated)
└── allocation_output.xlsx   ← final allocation report (auto-generated)
```

---

## Quick Start

### 1 — Install dependencies
```bash
pip install -r requirements.txt
```

### 2 — Prepare data

**Option A — Use your own Google Form export**
Copy your export as `responses.xlsx` (keep Google Form column order).
Create `config.xlsx` with columns: `Course Name | Group | Sections | Capacity | Prerequisites`.

**Option B — Generate sample data (for testing)**
```bash
python create_sample_data.py
```

### 3 — Run
```bash
python main.py
```

With custom paths:
```bash
python main.py --responses my_form_export.xlsx --config courses.xlsx --output result.xlsx
```

### 4 — Run Web Frontend
```bash
python web_app.py
```
Then open `http://127.0.0.1:5000`.

Web frontend features:
- Upload responses file (required)
- Upload config file (optional; auto-generated if omitted)
- Upload admin overrides file (optional)
- Choose allocation mode (`single`, `dual`, `auto`)
- Toggle section balancing
- Download output Excel, log file, and generated config per run

## Input: `responses.xlsx`

Expected columns (Google Form export):

| Column | Notes |
|---|---|
| Timestamp | Auto-added by Google Forms |
| Name | Student full name |
| Registration Number | Unique ID, e.g. `21BCS001` |
| Email | Student e-mail |
| Phone Number | Student phone |
| Section | Student's current class section (A / B / C) |
| CGPA | Accepts `8.9`, `8.90`, `89`, `89%` — all normalised to 0–10 |
| Courses Already Completed | Comma-separated or "None" |
| Select Primary Course (G1) | Top G1 elective choice |
| Select Secondary Course (G2) | Top G2 elective choice |

## Output: `allocation_output.xlsx`

| Sheet | Contents |
|---|---|
| Section A / B / C | Name, Reg No, CGPA, Primary Course, Secondary Course, Section |
| Allocated Students | Master list of all allocated/partially allocated students |
| Course Summary | Course, Group, Section, Enrolled, Capacity, Available, Fill % |
| Unallocated Students | Name, Reg No, Email, CGPA, Reason |

Row colour coding:
- **Blue header** — column labels
- **Amber row** — partial allocation (G1 only, no G2 assigned)
- **Red row** — fully unallocated
- **Orange row** — section 100 % full (Course Summary only)

---

## Allocation Rules

1. Students sorted by **CGPA ↓**, then **Timestamp ↑** (tie-breaker).
2. **G1 (Primary)**: iterate preferences → check prerequisites → check seats → assign first available section.
3. **G2 (Secondary)**: try to match G1 section → attempt section swap if full → fall back to majority section.
4. **Section balancing**: after all allocations, sections differing by > capacity/3 are rebalanced while preserving G1/G2 pairings.

```
[INFO] DL skipped for 21BCS007: missing prerequisite(s) {'ML'}

[INFO] Section swap: moved 21BCS010 from DS-A to DS-B (making room for 21BCS015)
[INFO] Section balance: moved 21BCS030 from AI-B to AI-A
```
